HLS Upload Limit Fix Audit - 2026-05-07

Summary

Reproduced the admin HLS upload failure as HTTP 413 from Kestrel's default 30000000 byte request limit. The app policy allows 200 MiB videos and nginx allows 250m, so the backend server limit was the first failing layer.

Changed

  • Configured Kestrel max request body size to video policy plus 1 MiB multipart overhead.
  • Configured multipart form body limit to the 200 MiB video policy.
  • Added a startup regression test for upload limit alignment.

Validation

  • RED focused startup test failed before implementation with Kestrel limit 30000000.
  • GREEN focused startup test passed after implementation.
  • Focused HLS integration tests passed 2/2.
  • WorkVideo policy unit tests passed 39/39.
  • Dev compose backend rebuild and health check passed.
  • Live 35 MiB repro changed from HTTP 413 to HTTP 400 app-level MP4 validation.
  • Backend solution tests passed: Contract 1 skipped, Component 126, Unit 59, Architecture 35, Integration 230.

Recommendation

Ship this fix before continuing performance work if production MP4 uploads larger than about 28.6 MiB are blocked. If failures continue after deployment, capture the exact HLS response body and investigate ffmpeg codec/transcode compatibility next.
