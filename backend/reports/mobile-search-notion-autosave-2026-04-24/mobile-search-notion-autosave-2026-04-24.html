Mobile Search / Notion Autosave Audit

Date: 2026-04-24

Changed

  • Mobile search now opens from the shared navbar and appears under the sticky header.
  • Page-level mobile search UI is hidden by default; desktop search stays inline.
  • Compact feeds no longer auto-append on scroll and now extend only via Load more.
  • Notion autosave shows Saving... earlier and uses a shorter debounce.
  • Repeated mutation session preflight now uses a short TTL cache.

Not Changed

  • Card preview surfaces still use excerpt-based summaries by design.
  • Desktop pagination behavior remains unchanged.
  • Notion autosave still uses the existing full blog update endpoint.

Validation

  • Targeted Vitest suite: pass
  • npm run typecheck: pass
  • npm run lint: pass with warnings only
  • Targeted Playwright suite against external Docker HTTP server: pass

Recommendation

Keep this pass and, if notion latency still matters, follow with a content-only autosave endpoint and server-side content parsing deduplication.
