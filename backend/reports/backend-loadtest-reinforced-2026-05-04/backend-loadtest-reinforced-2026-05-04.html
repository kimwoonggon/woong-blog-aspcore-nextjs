Scope

Expanded existing /admin/load-test observability in current Next.js + ASP.NET Core + nginx/compose architecture (no SignalR/k8s/NBomber additions).

Implemented

  • Time-based soak and ramp-based spike execution.
  • concurrency max raised to 1000 and clarified as in-flight HTTP requests.
  • Run status now shows current/peak in-flight and elapsed time.
  • HTTP failure breakdown includes 5xx/429/503/timeout/abort.
  • Backend diagnostics endpoint added at /api/admin/load-test/diagnostics (admin-only).
  • DB command/connection latency percentiles, slow query samples, timeout/error counts, and pg_stat_activity connection states.
  • Resilient fallback: diagnostics collector failures return database.status=error without endpoint crash.
